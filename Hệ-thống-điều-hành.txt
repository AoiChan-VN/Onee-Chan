```
            () -> cultivatorDAO.findByUUID(uuid)  [ASYNC: DB query]
          )
          .thenAccept(snapshot ->
            Bukkit.getScheduler().runTask(plugin, // [BACK TO MAIN]
              () -> {
                Cultivator c = Cultivator.fromSnapshot(snapshot);
                cache.put(uuid, c);
                // Now safe to use Bukkit API
              }
            )
          )

PlayerJoinEvent                              [MAIN thread]
  └→ cultivatorManager.getOrWait(uuid)      ← blocks 0 ticks if already cached
  └→ Send welcome message (MiniMessage)
  └→ effectManager.startAura(player)
```

**🧘 Meditation Flow:**
```
/tu meditate → TuSubCommand                  [MAIN thread]
  └→ meditationManager.startSession(uuid)
       └→ Validate: not already meditating, not in combat
       └→ Put MeditationSession record in activeSessions map
       └→ Send MiniMessage start notification

MeditationTask (runs every 20t SYNC):
  For each UUID in activeSessions:
    1. Verify player still online → remove if not
    2. Verify no movement since last tick → remove if moved
    3. expGain = FormulaEngine.calcMeditationExp(realm, stage, spiritRoot)
    4. CultivationEngine.grantExp(uuid, expGain)
         └→ Fire CultivationExpGainEvent (cancellable)
         └→ If event not cancelled: cultivator.addExp(expGain)
         └→ Check stage-up: if exp >= stageMax → processStageUp()
    5. EffectManager.tickMeditationAura(player)
```

**⚡ Breakthrough Flow:**
```
/tu breakthrough → BreakthroughEngine.validate()    [MAIN thread]
  ├→ FAIL: cultivator.stage < 9 → "Chưa đạt tầng 9"
  ├→ FAIL: cultivator.inBreakthrough == true → "Đang đột phá"
  ├→ FAIL: required items missing → "Thiếu vật phẩm"
  └→ PASS: Fire CultivationBreakthroughEvent (cancellable)

If not cancelled:
  └→ cultivator.inBreakthrough = true (atomic flag)
  └→ Does realm require Thiên Kiếp?
        YES → TribulationManager.startTribulation(uuid, TribulationData)
               └→ TribulationTickTask begins (sync, 20t)
               └→ TribulationEndEvent: SUCCESS
                    └→ BreakthroughEngine.execute(uuid)
                    └→ cultivator.inBreakthrough = false
               └→ TribulationEndEvent: FAIL
                    └→ TribulationEngine.applyPenalty(uuid)
                    └→ cultivator.inBreakthrough = false

        NO → BreakthroughEngine.execute(uuid) directly
               └→ realm++, stage=1, exp=0
               └→ Fire CultivationBreakthroughCompleteEvent
               └→ BreakthroughEffect.play(player)
               └→ AsyncAutoSave triggered immediately

All breakthrough results → ASYNC save via snapshot
```

**💾 Quit / Save Flow:**
```
PlayerQuitEvent                              [MAIN thread]
  1. meditationManager.stopSession(uuid)    ← cancel, save partial progress
  2. tribulationManager.abortTribulation(uuid) ← apply fail penalty
  3. CultivatorSnapshot snap = cultivator.toSnapshot()
  4. Bukkit.getScheduler().runTaskAsynchronously(plugin,
       () -> cultivatorDAO.upsert(snap)     [ASYNC: DB write]
     )
  5. cache.remove(uuid)                     ← Main thread, safe
  6. effectManager.stopAura(uuid)
```

## IV. THREAD SAFETY MECHANISMS

### Concurrent Collections (Main-thread writes, async reads for saves)

| Collection | Location | Type | Reason |
|---|---|---|---|
| `cache` | CultivatorManager | `ConcurrentHashMap<UUID, Cultivator>` | Multiple readers, load/unload writers |
| `pendingLoads` | CultivatorManager | `ConcurrentHashMap<UUID, CompletableFuture<CultivatorSnapshot>>` | Prevent double DB load on lag |
| `activeSessions` | MeditationManager | `ConcurrentHashMap<UUID, MeditationSession>` | MeditationTask reads from SYNC |
| `activeTribulations` | TribulationManager | `ConcurrentHashMap<UUID, TribulationState>` | Checked by listeners |
| `breakthroughLocks` | BreakthroughManager | `ConcurrentHashMap<UUID, AtomicBoolean>` | CAS to prevent duplicate breakthroughs |
| `sectCache` | SectManager | `ConcurrentHashMap<UUID, Sect>` | UUID = sect UUID |
| `pendingInvites` | SectManager | `ConcurrentHashMap<UUID, SectInvite>` | Invitee UUID → invite record |

### Critical Thread Rules (enforced via assertions)

```
RULE 1: Cultivator object mutated ONLY on main thread.
RULE 2: CultivatorSnapshot (record) used for ALL async DB ops.
RULE 3: AsyncPlayerPreLoginEvent → NO Bukkit API calls (different thread).
RULE 4: AutoSaveTask creates snapshot on SYNC tick, then saves ASYNC.
RULE 5: After any async operation returns, bridge back via:
         Bukkit.getScheduler().runTask(plugin, () -> { /* main thread */ })
RULE 6: DB connections NEVER held across tick boundaries.
RULE 7: All DAO methods are self-contained with try-with-resources.
```

## VI. DATABASE SCHEMA (SQLite — WAL Mode)

```sql
-- schema.sql (embedded in jar, applied by V1_InitialSchema.java)

PRAGMA foreign_keys = ON;
PRAGMA journal_mode = WAL;
PRAGMA synchronous = NORMAL;
PRAGMA cache_size = -8000;  -- 8MB cache

-- ─── CULTIVATORS ──────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS cultivators (
    uuid              TEXT    NOT NULL PRIMARY KEY,
    username          TEXT    NOT NULL,
    realm             TEXT    NOT NULL DEFAULT 'LUYEN_KHI',
    stage             INTEGER NOT NULL DEFAULT 1,
    cultivation_exp   INTEGER NOT NULL DEFAULT 0,
    spirit_root       TEXT    NOT NULL DEFAULT 'NGU_LINH',
    elements          TEXT    NOT NULL DEFAULT 'KIM,MOC,THUY,HOA,THO',
    spirit_energy     REAL    NOT NULL DEFAULT 100.0,
    spirit_energy_max REAL    NOT NULL DEFAULT 100.0,
    combat_power      INTEGER NOT NULL DEFAULT 0,
    meridians         TEXT    NOT NULL DEFAULT '000000000000',
    sect_id           TEXT             DEFAULT NULL,
    sect_rank         TEXT             DEFAULT NULL,
    last_online       INTEGER NOT NULL DEFAULT 0,
    created_at        INTEGER NOT NULL DEFAULT 0
);

CREATE INDEX IF NOT EXISTS idx_cultivators_sect ON cultivators(sect_id);

-- ─── SKILLS ───────────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS cultivator_skills (
    player_uuid  TEXT    NOT NULL,
    skill_id     TEXT    NOT NULL,
    skill_level  INTEGER NOT NULL DEFAULT 1,
    learned_at   INTEGER NOT NULL DEFAULT 0,
    PRIMARY KEY (player_uuid, skill_id),
    FOREIGN KEY (player_uuid) REFERENCES cultivators(uuid) ON DELETE CASCADE
);

-- ─── SECTS ────────────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS sects (
    id           TEXT    NOT NULL PRIMARY KEY,
    name         TEXT    NOT NULL UNIQUE,
    description  TEXT             DEFAULT NULL,
    leader_uuid  TEXT    NOT NULL,
    level        INTEGER NOT NULL DEFAULT 1,
    experience   INTEGER NOT NULL DEFAULT 0,
    created_at   INTEGER NOT NULL DEFAULT 0,
    announcement TEXT             DEFAULT NULL
);

-- ─── SECT MEMBERS ─────────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS sect_members (
    player_uuid  TEXT    NOT NULL PRIMARY KEY,
    sect_id      TEXT    NOT NULL,
    rank         TEXT    NOT NULL DEFAULT 'NGOAI_MON',
    joined_at    INTEGER NOT NULL DEFAULT 0,
    contribution INTEGER NOT NULL DEFAULT 0,
    FOREIGN KEY (player_uuid) REFERENCES cultivators(uuid) ON DELETE CASCADE,
    FOREIGN KEY (sect_id)     REFERENCES sects(id)         ON DELETE CASCADE
);

CREATE INDEX IF NOT EXISTS idx_sect_members_sect ON sect_members(sect_id);

-- ─── SCHEMA VERSION ───────────────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS schema_version (
    version    INTEGER NOT NULL PRIMARY KEY,
    applied_at INTEGER NOT NULL
);
```

## IX. EDGE-CASE & BUG PREVENTION MATRIX

| # | Danh Mục | Tình Huống | Cơ Chế Ngăn Chặn | Xử Lý Khi Xảy Ra |
|---|---|---|---|---|
| 1 | **Player** | Thoát khi đang tu luyện | `PlayerQuitEvent` → `meditationManager.stopSession()` | Lưu exp đã tích lũy trước khi xóa session |
| 2 | **Player** | Thoát khi đang Thiên Kiếp | `PlayerQuitEvent` → `tribulationManager.abortTribulation()` | Áp dụng penalty FAIL, reset `inTribulation=false` |
| 3 | **Player** | Thoát khi đang đột phá | `PlayerQuitEvent` → check `inBreakthrough` flag | Reset flag, KHÔNG cho breakthrough thành công |
| 4 | **Player** | Bị kick mid-operation | `PlayerKickEvent` → same as QuitEvent | Cùng logic với PlayerQuitEvent |
| 5 | **Player** | Chết trong Thiên Kiếp | `EntityDeathEvent` → detect if `inTribulation` | Trigger `TribulationEndEvent(FAIL)` → penalty |
| 6 | **Thread** | Double DB load (lag spike join) | `pendingLoads ConcurrentHashMap`; check before launching new Future | Trả về Future đang chờ, không tạo load mới |
| 7 | **Thread** | Async load xong sau khi player đã thoát | `thenAccept` kiểm tra `Bukkit.getPlayer(uuid) != null` | Discard kết quả, không cache |
| 8 | **Thread** | Bukkit API gọi từ async | `CultivationEngine` assert `Bukkit.isPrimaryThread()` ở đầu mọi method | Throw `IllegalStateException` với log chi tiết |
| 9 | **Thread** | Race condition trên `inBreakthrough` | `ConcurrentHashMap<UUID, AtomicBoolean>` với CAS `compareAndSet(false, true)` | Từ chối nếu CAS fail → "Đang xử lý yêu cầu" |
| 10 | **Database** | Connection timeout/lost | HikariCP tự reconnect; timeout 30s; max pool size 3 | Log WARN, skip save cycle, retry next interval |
| 11 | **Database** | Corrupt SQLite row (invalid enum) | `CultivatorDAO.fromResultSet()` dùng try-catch + fallback default | Log ERROR với UUID, load fresh default cultivator |
| 12 | **Database** | Migration fail | Transaction quanh mỗi migration; rollback on exception | Log FATAL, `plugin.setEnabled(false)` gracefully |
| 13 | **Config** | Missing config key | `ConfigManager.getOrDefault(path, hardcoded_default)` everywhere | Log WARN "Thiếu config [path], dùng default [value]" |
| 14 | **Config** | Giá trị âm/không hợp lệ | Validate range trong `ConfigManager`: `Math.max(min, value)` | Log WARN + clamp to minimum sensible value |
| 15 | **World** | World bị unload | Mọi location operation kiểm tra `world.isLoaded()` trước | Skip operation + log DEBUG |
| 16 | **World** | Block/Entity null return | `Optional<>` wrapping mọi `getTargetBlockExact()`, `getEntity()` | Early return, no NPE |
| 17 | **Item** | `getItemInMainHand()` → AIR | `ItemUtil.isAir(item)` guard trên mọi item listener | Ignore event, continue |
| 18 | **Item** | PDC read trên item corrupt | try-catch quanh `pdc.get(key, type)` | Return `Optional.empty()`, ignore item |
| 19 | **Item** | Inventory đầy khi nhận thưởng | `ItemGrantResult.droppedOnGround` flag | Drop at feet + notify player bằng MiniMessage |
| 20 | **Skill** | Cooldown bypass (lag exploit) | Server-side: `Map<UUID, Map<String, Long>>` với `System.currentTimeMillis()` | Reject và inform player "Kỹ năng đang hồi chiêu" |
| 21 | **Skill** | Spirit energy underflow | `Math.max(0, spiritEnergy - cost)` trước khi kích hoạt | Check `spiritEnergy >= cost` → từ chối nếu thiếu |
| 22 | **Sect** | Leader thoát game | Sect dùng `leaderUUID` (UUID), không phải `Player` ref | Sect vẫn hoạt động; Trưởng Lão có thể quản lý |
| 23 | **Sect** | Sect bị xóa khi member online | `SectDisbandEvent` notify tất cả online member | Clear `sectId` và `sectRank` trên mọi member's Cultivator |
| 24 | **Sect** | Leader UUID mồ côi (không có trong DB) | Validate trên startup: kiểm tra cultivators table | Log WARN, set leader = member rank cao nhất |
| 25 | **Sect** | Mời kép (double invite) | `pendingInvites ConcurrentHashMap` — put overwrites cũ | Chỉ 1 invite pending per invitee at a time |
| 26 | **Memory** | Cache không được clear | `PlayerQuitEvent` → `cache.remove(uuid)` + `onDisable()` clear all | `onDisable()` save + clear toàn bộ cache |
| 27 | **Memory** | Player object reference bị lưu lâu dài | **FORBIDDEN**: Never store `Player` in maps. Chỉ dùng `UUID` | Code review + `@NotNull UUID` enforced in all managers |
| 28 | **Memory** | World object reference bị lưu lâu dài | **FORBIDDEN**: Store `Position/UUID` not `World`. Re-fetch via `Bukkit.getWorld(uuid)` | Same as above |
| 29 | **Performance** | 100 player đang meditate | `MeditationTask` O(n) với lightweight per-entry logic | Mỗi entry chỉ tính exp + update 1 flag; < 0.01ms/player |
| 30 | **Performance** | EffectTask overhead | `EffectTickTask` mỗi 4t; skip nếu không có player trong radius | `O(1)` radius check trước khi spawn particles |
| 31 | **Performance** | `PlayerMoveEvent` spam | `MeditationListener` kiểm tra only if `cultivator.meditating == true` (O(1) volatile read) | Return immediately nếu không meditating |
| 32 | **Anti-exploit** | Breakthrough spam | `breakthroughLocks` AtomicBoolean; cooldown trong config | CAS fail → reject; ngăn concurrent breakthrough |
| 33 | **Startup** | Plugin load order | `paper-plugin.yml` declares PAPI/Vault `load: BEFORE` | Integrations check `Bukkit.getPluginManager().isPluginEnabled()` trước khi bind |
| 34 | **Shutdown** | Tasks không bị cancel | `TaskManager.cancelAll()` trong `onDisable()` | Mọi BukkitTask registered qua TaskManager |
